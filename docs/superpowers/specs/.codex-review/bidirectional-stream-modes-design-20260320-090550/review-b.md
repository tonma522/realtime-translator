[High] 「エラーハンドリング」は `direction_parse_failed` を不完全履歴として残して再翻訳だけ無効化しているが、返答アシスト・議事録生成・エクスポートから除外する仕様がないため、`translation=""` の履歴が下流機能に混入し、`既存の履歴...の前提を壊さない` という要件と矛盾している。  
[High] 「履歴」「データモデル変更」は `virtual_stream_id` / `resolved_direction` を追加する一方で、既存の `stream_id=listen/speak` をどこまで保持するか定義しておらず、履歴一覧・再翻訳・返答アシスト・議事録側の `listen/speak` 前提に対する互換契約が不足している。  
[Medium] 「設定ウィンドウの追加項目」「表示設計」は結果表示ラベルの更新しか規定しておらず、メイン画面の `聴く (PC音声→日本語)` / `話す (マイク→英語)`、状態表示、PTTボタン文言を選択モードにどう追従させるか未定義なので、`ja_en` や `auto` 選択時にUIが実際の動作方向と食い違う。  
[Medium] 「Whisper / 外部STT」「設定ウィンドウの追加項目」の `2フェーズを無効化または強制オフ` は挙動が曖昧で、旧設定に `two_phase_enabled=true` が残っている場合や設定画面でSTTバックエンドを切り替えた瞬間に、どこに理由を表示し、保存値をどう正規化するかが不足しており、運用上は「勝手にOFFになった」ように見えやすい。  
[Medium] 「音声チャンク受付」「エラーハンドリング」は PTT が `mic_mode` を共有すると書くだけで、録音中や送信直後に現在の固定方向/`auto` をどこへ表示するか、`auto` 判定失敗時にPTT利用者へどの再試行導線を見せるかが不足しており、PTTの操作感だけ不自然になりやすい。
