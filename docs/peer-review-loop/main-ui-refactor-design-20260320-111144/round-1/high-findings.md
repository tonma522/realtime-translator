# High Findings Summary

- [High] エラーを `ブロッカー` と `実行時エラー` に分けるだけで、キューイベント上の分類契約が未定義
- [High] パネル間更新 API の具体インターフェースと導入順序が未定義で、中間状態の二重管理リスクがある
- [High] 段階的実装の各ステップごとの回帰判定方法が未定義

## 修正に直結する要点

- `error` 系イベントの payload 契約と分類責務を明文化する
- `app.py` と各パネル間の更新 API を先に定義し、レイアウト移行前からその API に寄せる
- 各ステップの検証ゲートを追加し、中間状態でも回帰確認できるようにする
