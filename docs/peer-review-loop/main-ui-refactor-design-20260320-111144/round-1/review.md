## レビュー結果

---

### Findings（重要度順）

---

#### [高] 1. エラーイベントの分類方法が未定義

**仕様の抜け**

エラーを「ブロッカー」と「実行時エラー」に分けると明記しているが、キューイベント側でどう区別するかが未定義。`_poll_queue()` が受け取る `error` 系イベントのペイロード設計（種別フラグ？例外型で判断？）がなく、実装者が独自に判断せざるを得ない。

---

#### [高] 2. パネル間更新 API の具体性が低い

**バグ・回帰リスク**

「各パネルはメソッドまたは専用更新 API で情報を受け取る」とあるが、インターフェースの定義がない。段階的実装ではこれが最終ステップ（Step 6）のため、Steps 1〜5 の期間は既存の `tk.*Var` 直接参照と新 API が混在する。このオーバーラップ期間に設定値の二重管理・不整合が生じるリスクが高い。

---

#### [高] 3. 段階的実装の各ステップ完了時の回帰判定方法が未定義

**テスト不足**

Step 1〜6 を順に進める構成だが、各ステップの「完了＝既存機能が壊れていない」を何で確認するかが書かれていない。特に Step 2（タイムラインパネル独立）と Step 3（左カラム追加）の間で、状態バーや `_poll_queue()` の配線が中途半端な状態になりやすい。

---

#### [中] 4. 設定変更 → 左カラム要約の即時反映機構が未定義

**仕様の抜け**

「設定変更後に即反映される」と書かれているが、更新契機（ポーリング？ `trace` コールバック？ 設定ウィンドウ閉じ時のコールバック？）が未定義。`TranslatorApp` → 左カラム要約更新の矢印だけがデータフロー図に書かれており、実装根拠がない。

---

#### [中] 5. 状態バー優先順位ロジックの実装所在が曖昧

**仕様の抜け**

コンポーネント境界では `translation_timeline_panel` が「状態バー表示」責務を持つとされているが、優先順位（エラー > PTT録音中 > 翻訳中 > …）を解決するロジックの所在が `translation_timeline_panel` なのか `app.py` なのか明記されていない。複数状態が同時に立った場合の扱いも未定義。

---

#### [中] 6. 未解決事項3点が完了条件に含まれていない

**完了条件の不備**

仕様末尾で「実装着手前に確定対象に含める」と書きながら、完了条件にはそれらの解決を求める項目がない。結果として、以下が未定義のまま実装完了と判定できてしまう。

- 左カラムの方向サマリー／モード要約の粒度
- 開始前ブロッカーの表示 UI 形式（ラベル or カード）
- 右タブの最小幅下限

---

#### [中] 7. 複数エラー同時発生時のテストが未記載

**テスト不足**

状態バーの優先順位は定義されているが、「API キー未設定（ブロッカー）+ 翻訳中に単発失敗（実行時エラー）」が同時に存在する場合のテストシナリオがない。優先順位ロジックの境界ケースを確認する手段がなく、仕様通りの動作保証ができない。

---

#### [低] 8. 補助タブ切替中に翻訳イベントが届いた場合のテストが未記載

**テスト不足**

`translation_done` イベントが `workspace_panel` に届く際、ユーザーが別タブを開いていた場合のタブ状態・スクロール位置・結果反映の動作が未定義でありテストにも含まれていない。

---

#### [低] 9. 完了条件「機能が退行していない」が定性的すぎる

**完了条件の不備**

翻訳・再翻訳・返答アシスト・議事録が退行していないことを、どのシナリオで確認するかが書かれていない。テスト方針のチェックリストとの対応が取れておらず、完了判断が属人的になる。

---

#### [低] 10. キーボード操作のテストが未記載

**テスト不足**

PTT がキーボードトリガーである場合の操作確認、またはタブ切替のキーボード操作が既存通り機能するかの検証が含まれていない。

---

### 重大指摘

あり（上記 1・2・3 が特に重大）

---

### 総評

UI の役割分担と優先順位の考え方は明快で、段階的実装の順序も妥当。ただし、エラーイベントの分類方法とパネル間更新 API の設計が未定義のまま実装に入ると、分割リファクタの中間状態でデータ不整合や配線取りこぼしが顕在化しやすい。未解決事項3点を「着手前に確定」と書きながら完了条件に含めていない点は、検収時の判断基準を曖昧にするため、完了条件への明示的追加を推奨する。
